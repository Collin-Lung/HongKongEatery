Example "industry standard" directory structure for modern web development projects


/project-root
│
├── /public                # Public assets served directly without processing
│   ├── index.html         # Main HTML file (entry point)
│   ├── favicon.ico        # Site favicon
│   ├── logo.png           # Static assets (images, fonts, etc.)
│   └── robots.txt         # SEO and crawler-related files
│
├── /src                   # Source code for development
│   ├── /assets            # Assets (images, fonts, etc.) for use in development
│   │   └── styles         # Custom CSS files (if any, aside from Tailwind)
│   │       └── main.css   # Entry point CSS that imports Tailwind or custom styles
│   │
│   ├── /components        # Reusable UI components (HTML, JSX, Vue, etc.)
│   │   ├── header.html    # Example: A header component
│   │   └── footer.html    # Example: A footer component
│   │
│   ├── /pages             # Pages of the site (can be HTML or JS/JSX files)
│   │   ├── home.html      # Example: Home page
│   │   └── about.html     # Example: About page
│   │
│   ├── /scripts           # JavaScript or TypeScript files
│   │   └── app.js         # Main JavaScript file
│   │
│   └── /styles            # Tailwind and other custom styles
│       └── tailwind.css   # Tailwind entry point (imported in main.css)
│
├── /dist                  # Distribution folder for production build (generated)
│   └── (output files)
│
├── /node_modules          # Node.js modules (auto-generated by npm or yarn)
│
├── tailwind.config.js      # Tailwind CSS configuration file
├── postcss.config.js       # PostCSS configuration (if using PostCSS)
├── package.json            # Project metadata and dependencies
├── package-lock.json       # Dependency lock file (generated by npm/yarn)
├── webpack.config.js       # Webpack or Vite config (if using a bundler)
└── .gitignore              # Git ignore file (e.g., ignore node_modules, dist, etc.)
